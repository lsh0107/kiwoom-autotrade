# 코드 리뷰

$ARGUMENTS 에 대한 코드 리뷰를 수행하세요.

## 리뷰 체크리스트

### 1. 정확성
- [ ] 로직이 의도대로 동작하는가
- [ ] 엣지 케이스 처리가 되어 있는가
- [ ] 에러 처리가 적절한가

### 2. 보안 (트레이딩 시스템 특화)
- [ ] API 키/비밀번호가 하드코딩되지 않았는가
- [ ] 주문 금액/수량에 안전 제한이 있는가
- [ ] 모의투자/실거래 구분이 명확한가
- [ ] 입력 검증이 충분한가

### 3. Python 코드 품질
- [ ] type hint가 적용되어 있는가
- [ ] 함수/클래스에 docstring이 있는가
- [ ] 네이밍 컨벤션 (snake_case) 준수
- [ ] 불필요한 복잡성이 없는가

### 4. 아키텍처
- [ ] 단일 책임 원칙을 따르는가
- [ ] 적절한 추상화 수준인가
- [ ] 테스트 가능한 구조인가

### 5. 성능
- [ ] 불필요한 API 호출이 없는가
- [ ] 적절한 캐싱/배치 처리가 되어 있는가
- [ ] 메모리 누수 가능성이 없는가

## 출력 형식
심각도별로 분류하여 보고:
- **CRITICAL**: 즉시 수정 필요 (보안, 데이터 손실 위험)
- **WARNING**: 수정 권장 (버그 가능성, 코드 품질)
- **INFO**: 개선 제안 (가독성, 성능 최적화)
