### 19:30 - [lead] Phase 1 리스크 관리 설계 검증 — 6개 이슈 토론

리드가 design-phase1-risk-management.md 설계 문서를 코드/리서치와 교차 검증하여

#### 이슈 #1: ATR 동적 손절에 최소 바닥값 없음

**문제**: ATR% 작은 종목(예: 0.2%)에서 `dynamic_stop = -(0.2% × 1.5) = -0.3%`.
왕복 거래비용 0.23% → 손익분기 최소 0.46% 필요 → **거래할수록 손실**.

**리서치 결과** (Kevin Davey, KJ Trading Systems / LuxAlgo / Leung & Li 2015):
- 전문 시스템은 `max(ATR×multiplier, fixed_minimum)` 패턴 사용 (floor/ceiling)
- 바닥 0.5%가 한국 시장 거래비용(0.23%) 감안 시 적정
- **더 나은 방법**: ATR% < 0.35% 종목은 아예 진입 스킵 (변동성 필터)
  - 근거: 0.35% × 1.5 = 0.525% > 0.46% (손익분기 통과)
  - ATR이 너무 작은데 0.5% 바닥을 강제하면 SL이 자연 변동폭의 2배 → 거의 안 걸림 → 사실상 미관리 포지션

**결정**:
1. 바닥: `dynamic_stop = -max(ATR% × 1.5, 0.005)`
2. 필터: `ATR% < 0.35% → 진입 스킵` (바닥보다 필터가 우선)
3. TP 바닥: `dynamic_tp = max(ATR% × 3.0, 0.010)` (R:R 1:2 유지)

#### 이슈 #2: R:R 비율 1:3 → 1:2 변경 미분석

**문제**: 고정 SL/TP는 R:R 1:3 (-0.5%/+1.5%), 동적은 R:R 1:2 (1.5×/3.0×ATR).
design-strategy-v2.md에는 4.5×ATR (1:3)으로 적혀있어 문서 간 불일치.

**수학적 분석** (거래비용 0.23% 포함 손익분기 승률):

| R:R | SL | TP | 비용 無 | 비용 有 |
|-----|----|----|--------|--------|
| 1:3 (고정) | 0.5% | 1.5% | 25.0% | **36.5%** ← 비용 타격 극심 |
| 1:3 (ATR 1.6%) | 2.4% | 7.2% | 25.0% | **27.4%** |
| 1:2 (ATR 1.6%) | 2.4% | 4.8% | 33.3% | **36.5%** |
| 1:2 (ATR 1.3%) | 1.95% | 3.90% | 33.3% | **37.3%** |

핵심: 고정 0.5%/1.5%는 거래비용 영향이 +11.5pp로 치명적. ATR 기반은 +2~4pp로 합리적.

**한국 시장 특성** (KAIST 일중모멘텀 연구, Tandfonline 2024):
- 한국 시장 일중 모멘텀은 미국보다 **약함** (weak evidence)
- 4.5×ATR (예: +7.2%) 목표가를 하루 안에 달성하기 거의 불가능
- HPSP 백테스트: R:R 1:2에서 PF 4.12, R:R 1:3에서 PF 3.23 → **1:2가 우수**
- 먼 TP는 승률 하락으로 상쇄됨 (75% → 67%)

**결정**:
1. **R:R = 1:2 확정** (SL=1.5×ATR, TP=3.0×ATR)
2. design-strategy-v2.md의 4.5×ATR → 3.0×ATR 수정
3. ATR 미산출 시 → **진입 스킵** (고정값 폴백 사용 안 함)
   - 근거: R:R 혼재(동적 1:2 / 고정 1:3)는 기대값 계산을 불안정하게 만듦
   - 최후 수단 폴백이 필요하다면 고정 SL=-2%, TP=+4% (R:R 1:2 통일)

#### 이슈 #3: 2연패 블랙리스트 vs 포지션 축소 모순

**문제**: 문서가 "알고에는 쿨다운 불필요, 포지션 축소가 낫다"(Connors 인용)고 하면서
바로 옆에서 2연패 시 블랙리스트(= 쿨다운)를 적용. 자기 모순.

**리서치 결과** (Connors 원문, Kevin Davey, LuxAlgo):
- **Connors 인용이 잘못됨**: Connors는 **평균회귀** 전략에 대해 말한 것.
  - 평균회귀: 가격이 역행할수록 반등 확률 상승 → 쿨다운하면 기회 상실
  - 모멘텀: 연속 손실 = 해당 종목에 모멘텀이 없다는 증거 → 스킵이 합리적
- **Kevin Davey**: 연속 손실 후 전략 중단은 **역추세 전략**에서 유효. 모멘텀에서는 다른 논리.
- **전문 시스템 패턴**: 포트폴리오 레벨 사이징 + 종목별 시그널 품질 필터 병행

**핵심 전환**: 블랙리스트를 "쿨다운"이 아니라 **"시그널 품질 필터"**로 재정의.
- "이 종목에서 모멘텀 전략이 3번 연속 실패 = 오늘 이 종목에 모멘텀이 없다"
- 이건 감정 관리가 아니라 **시그널 유효성 판단**

**결정**:
1. 단계적 접근:
   - 1연패 → 변경 없음 (정상 분산)
   - 2연패 → 포지션 사이즈 50% 축소 (anti-martingale)
   - 3연패 → 당일 블랙리스트 (시그널 품질 필터: "오늘 이 종목에 모멘텀 없음")
2. Connors 인용 삭제 (평균회귀 전용, 모멘텀과 무관)
3. 근거를 "시그널 품질 기반"으로 변경

#### 이슈 #4: force_close_time 3곳 불일치

**문제**: 설계문서 15:25, 코드 14:00, 주석 14:30. 어디가 맞는지 불명확.

**리서치 결과**:
- **RC4058 에러**: 실제 로그 분석 결과 15:30:38에 최초 발생 → 모의투자 장종료 시점
  - 설계문서의 "마지막 5분" 설명은 부정확
- **15:20~15:30**: 동시호가(closing auction) → 주문 즉시 체결 안 됨, 가격 통제 불가
- **한국 자동매매 관행**: 15:10~15:15에 청산이 일반적
- **시간대별 분석**:
  - 14:00: 너무 이름 — 12:30 진입 포지션이 1.5시간만에 강제 청산
  - 14:30: 보수적 — 마감 모멘텀(14:30~15:20) 활용 불가
  - 15:15: 최적 — 마감 모멘텀 캡처 + 동시호가 5분 전 안전 마진
  - 15:25: 위험 — 동시호가 구간, 가격 통제 불가

**결정**:
1. **force_close_time = 15:15 통일** (모든 위치)
2. 실패 시 15:18 재시도 (공격적 지정가), 15:19까지 미체결 시 동시호가 주문
3. 변경 위치: MomentumParams, live_trader.py, grid_search.py, 문서 전체

#### 이슈 #5: risk_scale 복구 로직 없음

**문제**: daily_pnl ≤ -2% → risk_scale=0.5 설정 후, 수익으로 회복해도 0.5 고정.

**코드베이스 탐색 결과**:
- `kill_switch.py`에 이미 구현된 것들:
  - `DRAWDOWN_STOP_BUY_PCT = -2%`, `DRAWDOWN_FORCE_CLOSE_PCT = -3%`
  - `update_drawdown()`, `update_weekly_loss()` 함수 존재
  - 일일 자정 자동 리셋 로직 존재
- **하지만** live_trader.py에서 이 함수들을 **한 번도 호출하지 않음**
- PR-C의 TradingState 필드가 kill_switch의 KillSwitchState와 **중복**

**결정**:
1. **PR-C는 kill_switch.py와 통합** — TradingState에 중복 필드 추가하지 않음
   - `symbol_losses`, `symbol_blacklist`만 TradingState에 추가 (kill_switch에 없는 것)
   - `daily_pnl`, `risk_scale`은 kill_switch의 기존 메커니즘 활용
2. **risk_scale 복구 안 함 (의도적)**
   - 근거: 일일 -2% 도달 = 오늘 시장 조건이 전략에 불리하다는 증거
   - 일시적 회복 후 재하락 시 더 큰 손실 위험
   - 다음날 자정에 자동 리셋 (kill_switch의 기존 daily reset)
3. 문서에 명시: "당일 -2% 달성 시 장 종료까지 0.5 유지. 다음날 자동 리셋."
