## 1. DART 전자공시 API (dart.fss.or.kr)

| 항목 | 내용 |
|------|------|
| **API URL** | `https://opendart.fss.or.kr/api/` |
| **인증** | API 키 (무료 발급, opendart.fss.or.kr 신청) |
| **Rate Limit** | 분당 1,000회 초과 시 제한 (dart-fss 문서 기준) |
| **무료/유료** | 완전 무료 |
| **데이터 형식** | JSON, XML |
| **실시간/배치** | 배치 (공시 발생 시 갱신) |
| **수집 가능 여부** | 가능 (공식 Open API) |

**제공 데이터**:
- 정기공시: 사업보고서, 반기/분기보고서
- 주요사항보고서: 유상증자, 전환사채, 자기주식 취득 등
- 재무제표: XBRL 형식
- 지분공시, 대량보유보고

**파이썬 라이브러리**:
- `opendartreader`: `pip install opendartreader` — 공시 목록, 기업정보, 재무제표, 지분공시 전반
  - 주요 메서드: `list()`, `company()`, `finstate()`, `major_shareholders()`
- `dart-fss` v0.4.15 (MIT): `pip install dart-fss` — 재무제표 엑셀 추출 특화, 분당 1,000건 제한

**권장**: `opendartreader` 우선, 재무제표 정밀 추출 시 `dart-fss` 병행.

## 3. 한국거래소 (KRX)

| 항목 | 내용 |
|------|------|
| **API URL** | 공식 REST API 없음 (data.krx.co.kr 웹 다운로드 또는 pykrx 스크래핑) |
| **인증** | 없음 |
| **Rate Limit** | 명시 없음, "무분별한 호출 자제" 권고 (1초+ 간격 권장) |
| **무료/유료** | 무료 |
| **데이터 형식** | DataFrame (pykrx) |
| **실시간/배치** | 배치 (장 마감 후 갱신) |
| **수집 가능 여부** | 가능 (비공식 스크래핑, 실사용 검증됨) |

**pykrx v1.2.4 제공 데이터** (Python 3.10+, MIT):
- 주가 OHLCV (일별/기간별)
- 펀더멘탈: PER, PBR, EPS, BPS, DIV
- 투자자별 매매동향: 기관/개인/외국인 거래량·거래대금·순매수
- 지수 OHLCV 및 구성종목
- ETF, ETN, ELW 데이터
- 채권 장외 수익률
- 공매도 현황

**주의**: 비공식 스크래핑 기반. KRX 웹사이트 구조 변경 시 중단 가능. 호출 간격 1~2초 유지.

## 우선순위 요약

### Tier 1 — 즉시 적용 (공식, 무료, 안정적)

| 순위 | 소스 | 라이브러리 | 제공 데이터 | 수집 주기 |
|------|------|-----------|-----------|----------|
| 1 | **DART Open API** | `opendartreader` | 공시, 재무제표, 지분공시 | 일 1회 (장 마감 후) |
| 2 | **pykrx** | `pykrx` | 주가 OHLCV, PER/PBR, 투자자 매매 | 일 1회 (장 마감 후) |
| 3 | **FRED API** | `fredapi` | VIX, 미국 금리, WTI, USD/KRW | 일 1회 |
| 4 | **한국은행 ECOS** | `requests` 직접 | 기준금리, 거시경제 지표 | 주 1회 |

### Tier 2 — 조건부 적용 (비공식 또는 제약 있음)

| 순위 | 소스 | 라이브러리 | 비고 |
|------|------|-----------|------|
| 5 | **yfinance** | `yfinance` | 해외 지수 실시간, 비공식 ToS 주의 |
| 6 | **네이버 뉴스 검색 API** | `requests` | 종목 뉴스, 일 25,000건, 1~2시간 간격 |
| 7 | **네이버 데이터랩** | `requests` | 검색 트렌드, 일 1,000건 |

### Tier 3 — 보류/제외

| 소스 | 제외 이유 |
|------|----------|

## 수집 아키텍처 제안

```
[Tier 1 — 공식 API, 배치]
  DART         → 공시/재무제표       → 일 1회 (장 마감 17:00 이후)
  pykrx        → 주가/투자자 동향    → 일 1회 (장 마감 16:00 이후)
  FRED         → 매크로 지표        → 일 1회 (미국장 마감 후)
  ECOS         → 한국 거시경제      → 주 1회 (발표 스케줄 기반)

[Tier 2 — 보조, 실시간/빈번]
  yfinance     → 해외 지수 실시간   → 장 중 5분 간격
  네이버 뉴스   → 종목 뉴스         → 1~2시간 간격
```

**Why**: Tier 1은 공식 API로 법적 리스크 없고 안정적. Tier 2는 실사용 검증됐으나 모니터링 필요.
**How to apply**: Phase 3 Airflow DAG 설계 시 이 우선순위 기준으로 수집 스케줄러 구성.
