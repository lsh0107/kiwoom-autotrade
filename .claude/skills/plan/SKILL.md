# 설계 모드

$ARGUMENTS 에 대한 설계를 수행하세요.

## 설계 프로세스

### 1단계: 요구사항 분석
- 사용자 요구사항을 명확히 정리
- 기존 코드/아키텍처와의 관계 파악
- 제약 조건 식별 (키움 API 제한, 보안, 성능 등)

### 2단계: 기존 코드 탐색
- 관련 기존 코드를 Glob, Grep, Read로 탐색
- 영향 받는 모듈/파일 파악
- 기존 패턴과 컨벤션 확인

### 3단계: 설계안 작성
다음 항목을 포함하여 설계안을 제시:

- **목표**: 달성하려는 것
- **접근 방식**: 선택한 방식과 이유, 대안과 기각 이유
- **구현 계획**: 파일/모듈 단위 변경 사항
- **API/인터페이스**: 주요 클래스/함수 시그니처, 데이터 흐름
- **테스트 계획**: 단위/통합 테스트 범위
- **보안 고려사항**: API 키 관리, 주문 실행 안전장치
- **리스크**: 알려진 리스크와 대응 방안

### 4단계: 사용자 확인
- 설계안을 사용자에게 제시하고 승인 대기
- 승인 후 구현 시작

## 규칙
- 설계 시 코드를 작성하지 말 것 (탐색/읽기만)
- 반드시 기존 코드를 먼저 확인할 것
- 트레이딩 관련 기능은 반드시 안전장치(dry-run, 금액 제한) 포함
